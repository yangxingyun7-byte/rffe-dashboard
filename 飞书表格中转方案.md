# 飞书表格中转到 Dashboard 方案

## 目标

让 RFFE Dashboard 使用飞书表格作为上游数据源。

飞书表格地址：

```text
https://mi.feishu.cn/sheets/ICOts6o33hGMwmtJmSEcYY51nVf?sheet=902rKA
```

目标 sheet：

```text
RFFE Case control table
```

## 为什么不直接在前端读取飞书链接

当前 Dashboard 部署在 GitHub Pages，是纯静态网页。

直接在浏览器里访问飞书表格链接时，未登录会跳转到飞书登录页：

```text
https://accounts.feishu.cn/accounts/page/login...
```

因此前端无法稳定执行：

```ts
fetch("https://mi.feishu.cn/sheets/...")
```

原因包括：

1. 飞书表格需要登录态或 API token。
2. GitHub Pages 前端不能安全保存飞书 token。
3. 浏览器跨域和飞书登录重定向会阻止直接读取单元格数据。

## 当前采用方案

采用“本机/服务端中转”方案：

```text
飞书表格
  → 已授权 Feishu API 读取 sheet range
  → 生成 public/data/control-table.json
  → npm run build
  → 发布到 GitHub Pages
  → Dashboard 每 1 分钟读取 JSON
```

当前中转文件：

```text
C:\RFFE_DASHBOARD\rffe_proj\public\data\control-table.json
```

线上访问路径：

```text
https://yangxingyun7-byte.github.io/rffe-dashboard/data/control-table.json
```

## 前端读取方式

Dashboard 默认数据源已改为：

```text
/rffe-dashboard/data/control-table.json
```

前端 Hook：

```text
lib/useRffeData.ts
```

现在支持两种数据：

1. `.json`：读取飞书中转数据。
2. `.xlsx`：保留原 Excel 读取能力。

JSON 格式：

```json
{
  "source": "feishu",
  "spreadsheetToken": "ICOts6o33hGMwmtJmSEcYY51nVf",
  "sheetId": "902rKA",
  "sheetTitle": "RFFE Case control table",
  "updatedAt": "2026-06-23T16:30:00+08:00",
  "values": []
}
```

其中 `values` 是二维数组，行列结构保持和原 Excel 一致。

## 手动刷新中转数据

当前可在 Claude 会话中通过 Feishu MCP 读取：

```text
sheet_ops read
spreadsheet_token: ICOts6o33hGMwmtJmSEcYY51nVf
range: 902rKA!A1:Y23
render_option: FormattedValue
```

然后把返回值转换为纯文本二维数组，写入：

```text
public/data/control-table.json
```

再执行：

```powershell
cd C:\RFFE_DASHBOARD\rffe_proj
.\publish-github-pages.bat
```

## 后续自动化建议

如果要完全自动化，需要新增一个本机或云端脚本，直接调用飞书开放平台 API：

1. 读取飞书应用凭证。
2. 获取 `tenant_access_token` 或 `user_access_token`。
3. 调用电子表格读取接口。
4. 把目标 sheet range 转成 `public/data/control-table.json`。
5. 执行构建和发布脚本。

不建议把飞书 token 写入前端代码或 Git 仓库。

## 当前限制

1. 当前 JSON 是一次性导出的快照。
2. 飞书表格后续变化不会自动进入 Dashboard，除非再次导出 JSON 并发布。
3. 若要每分钟实时读取飞书，需要部署一个安全后端代理或本机定时导出任务。
